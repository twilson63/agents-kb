# Module 2: Skills & Memory

**Duration:** 50 minutes
**Goal:** Students leave knowing how to teach their agent about their world — who they are, how they work, and what they need — so every session starts from an informed baseline instead of zero.

---

## Essay 1 — The Blank Slate Problem

**Core idea:** Every new session with an AI agent starts from nothing. It doesn't know who you are, what you do, or how you like to work. Memory changes that.

**Outline:**
- Why agents forget: how sessions work
- The frustration of re-explaining yourself every time
- What it would look like if your agent actually knew you
- Memory as the solution: teaching your agent your context once so it applies it always
- Preview: two types of knowledge — who you are (memory) and how to do things (skills)

**Pod exercise:** Student lists three things they wish their agent already knew about them. This list becomes the foundation of their first memory note.

---

## Essay 2 — What Memory Actually Is

**Core idea:** Memory isn't magic — it's just structured notes your agent reads at the start of every conversation. Understanding this demystifies it and makes it easy to use.

**Outline:**
- Memory as a file your agent reads before it responds
- The difference between short-term context (in a conversation) and long-term memory (persisted)
- What belongs in memory: facts, preferences, recurring context, standing instructions
- What doesn't belong: one-off tasks, temporary information
- How to think about memory as a living document you update over time

**Pod exercise:** Student opens their practice pod's memory file and writes three memory notes: one about their role, one about their communication style, one about a recurring task they do.

---

## Essay 3 — Writing Your First Memory Note

**Core idea:** A good memory note is specific, factual, and useful across many conversations. This essay teaches students how to write ones that actually help.

**Outline:**
- The anatomy of a useful memory note: what it is, why it matters, when to apply it
- Bad memory note vs. good memory note (examples)
- Categories: personal context, team context, project context, preferences
- Keeping notes short and scannable — the agent reads these, not just you
- The update habit: memory notes should evolve as your situation changes

**Pod exercise:** Student writes five memory notes using the template provided. Instructor reviews a few examples with the group.

---

## Essay 4 — What Is a Skill?

**Core idea:** If memory is what your agent knows about you, skills are what it knows how to do for you. A skill is a reusable set of instructions for a recurring task.

**Outline:**
- The definition: a skill is a saved workflow your agent can follow on demand
- Everyday examples: writing in your voice, formatting reports your way, following your review process
- The difference between a skill and a prompt: skills are reusable, prompts are one-off
- Why skills matter: consistency, speed, quality — you only figure out the best way to do something once
- Skills as institutional knowledge: your team's collective know-how, codified

**Pod exercise:** Student identifies one task they do repeatedly that always requires the same type of output. They write one sentence describing what a skill for it would do.

---

## Essay 5 — Writing Your First Skill

**Core idea:** Writing a skill doesn't require technical knowledge. It's instructions in plain language — the same thing you'd tell a new colleague.

**Outline:**
- The structure of a skill: name, purpose, when to use it, step-by-step instructions
- Writing instructions as if you're briefing a smart person who's never done it
- Common first skills: weekly summary, email response in my voice, meeting notes format
- Testing your skill: does the agent do what you expected?
- Iterating: skills improve the more you use and refine them

**Pod exercise:** Student writes a complete skill using the template. Then activates it in their practice pod and runs it on a real example. Refine based on results.

---

## Essay 6 — Personal Context vs. Task Context

**Core idea:** Not all context is the same. Knowing the difference between who you are (personal) and what you're doing right now (task) helps you know what belongs in memory and what belongs in the conversation.

**Outline:**
- Personal context: stable facts about you that apply across everything (role, preferences, team, communication style)
- Task context: specific to the current work (this project, this deadline, this audience)
- What happens when you mix them up (memory gets cluttered, task context goes stale)
- Rule of thumb: if you'd tell a new colleague once and expect them to remember, it's memory. If you'd say it every time you assign a task, it's prompt context.
- Applying this: how to audit your memory notes for what belongs and what doesn't

**Pod exercise:** Student reviews the memory notes they wrote earlier and categorizes each as personal or task context. Move any task-specific notes out of memory.

---

## Essay 7 — Building a Skill Library Over Time

**Core idea:** The real value of skills compounds. The more you build, the more your agent feels like it was made for your specific work.

**Outline:**
- Starting small: one or two skills is enough to feel the difference
- The habit: every time you repeat a task, ask yourself "could this be a skill?"
- Organizing your skills: naming conventions, categories
- Sharing skills with your team: one person writes it, everyone benefits
- Skills as onboarding: new team members get your best practices on day one
- The long game: a skill library that reflects how your team actually works

**Pod exercise:** Student sketches a skill library map — three categories of skills they'd want to build over the next month. Share with the group.

---

## Essay 8 — When Memory and Skills Change Everything

**Core idea:** A closing essay that ties memory and skills together and shows students the compounding effect — what working with a well-configured agent actually feels like vs. starting from scratch every time.

**Outline:**
- Before and after: a day without memory/skills vs. a day with them
- The feeling of an agent that knows you: less explaining, faster results, better outputs
- Memory and skills as your personal AI configuration — your fingerprint on the tool
- The investment mindset: time spent configuring compounds into time saved forever
- Preview for Module 3: memory and skills become even more powerful when you delegate to multiple agents

**Pod exercise:** Student runs a full task end-to-end using their memory notes and at least one skill they built today. Debrief: what felt different from Module 1?

---

## Instructor Notes

- Many students are surprised memory is just text files — demystify this early with a quick demo
- Essay 5 (writing your first skill) is where students need the most time — budget for questions
- The "tell a new colleague once" rule of thumb for memory vs. prompt context is very intuitive for non-technical audiences
- Encourage students to think about their actual job throughout — memory and skills are only useful if they're grounded in real work
